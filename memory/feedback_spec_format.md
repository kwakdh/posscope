기획서를 만들어달라는 요청이 오면 아래 포맷을 반드시 따른다.

**Why:** 곽다희가 해당 포맷을 표준 기획서 형식으로 확정함.

---

## 기획서 종류별 규칙

### ① 파트너 기획서 (백오피스)
- **피그마 파일**: `sefEzyY60eu2B4W2kH6d0P`, 페이지: `　📁 파트너`
- **프레임 이름**: `[D-] [파트너] 기능명 수정` 또는 `[D-] [파트너] 기능명 추가` (항상 자동으로 붙임)
- **헤더 구성**: 브레드크럼 + 대제목 + 서브타이틀 + **적용 사용자 타입 pill** 포함
- **제목 형식**: `기능명 수정` / `기능명 추가` (기획서 X)

### ② 포스앱 기획서 (KCP POS 앱)
- **피그마 파일**: `sefEzyY60eu2B4W2kH6d0P`, 해당 기능의 페이지 (예: `APP_06_9 기본 설정`)
- **프레임 이름**: `[D-] 기능명 기능 수정` 또는 `[D-] 기능명 기능 추가` ([파트너] 없음)
- **헤더 구성**: 브레드크럼 + 대제목 + 서브타이틀만 — **적용 사용자 타입 pill 없음**
- **제목 형식**: `기능명 기능 수정` / `기능명 기능 추가` (기획서 X)

## 공통 포맷 구성 요소

### 헤더
- 브레드크럼 바: `#F5F5F7` 배경, `#6E6E73` 텍스트 (소형)
- 대제목: `#1D1D1F`, Bold, 26px
- 서브타이틀: 담당자·날짜, `#6E6E73`

### SCREEN 섹션
- SCREEN chip: `#0071E3` 파란 배경 + 흰 텍스트
- 화면 설명 텍스트: 좌측 여백(paddingLeft 40) 필수
- 좌측: 화면 캡처 이미지
- 우측: Descriptions 패널 (No / 항목 / 설명)

### UI 참고사항 메모 박스
"UI 참고사항 메모 추가해줘" 요청 시 아래 스타일로 Figma에 삽입:
- 위치: 헤더 구분선 바로 아래, SCREEN 01 앞
- 배경: `#FFF0F5` / 테두리: `#FF3B8E` 점선(dashPattern [6,4]) 1.5px / 모서리 8px
- 헤더: `★ UI 참고사항` Bold 13px `#FF3B8E`
- 내용: Regular 12px `#333333`, 불릿(•) 형식
- 래퍼: paddingLeft/Right 40, paddingTop/Bottom 16, 흰 배경
- Figma 코드: `memo.dashPattern = [6, 4]`로 점선 처리

### 연관 기획서 링크
- **연관된 두 기획을 동시에 진행하는 경우**, 완료 전에 반드시 **"두 기획서 간 연계 링크를 넣을까요?"** 물어볼 것
- 단일 기획서 작성 완료 후에도 다른 기획서와 연관이 있다면 물어볼 것
- 링크 추가 시: 양쪽 기획서 헤더에 상호 링크 텍스트 삽입 + `setRangeHyperlink`으로 Figma URL 연결
- 링크 텍스트 형식: `[파트너] 기능명 →` / `[포스앱] 기능명 →` (파란색 `#0071E3`, Semi Bold 11px)

### UI 변경 표시 — 두 가지 타입

#### ① 앱 화면 변경 (UI팀 요청용) — 분홍
- 앱 화면(포스앱/베리포스/파트너)에서 변경되는 영역에 사용
- 색상: `#FF3B8E` (분홍), 점선 border 2.5px, radius 10px
- 라벨: `UI작업 - 섹션 추가` / `UI작업 - 화면 변경` / `UI작업 - 메뉴 추가` 등

#### ② 영수증/인쇄물 영역 추가 — 초록
- 종이영수증·중간계산서 등 인쇄물에서 새로 추가되는 영역에 사용
- 색상: `#34C759` (초록), 점선 border 2.5px, radius 10px
- 라벨: `영역 추가` (UI작업 prefix 없음)

### 캡처 이미지 UI 상태 규칙 (필수)
- 디스크립션에 **"비활성화"** 가 있으면 해당 항목은 캡처본에서 반드시 비활성화 UI로 표현 (opacity 낮춤, 회색 처리 등)
- 디스크립션 상태 설명이 있으면 캡처본 UI 상태도 일치해야 함 (선택됨/비선택, ON/OFF 등)

### 완료 전 더블체크 (필수)
- 캡처본 배지 번호와 디스크립션 No 번호가 1:1로 정확히 대응하는지 **항상 마지막에 확인**
- 배지가 없는 항목이 디스크립션에 있거나, 반대로 배지가 있는데 디스크립션이 없으면 수정

### 캡처 이미지 규칙 (필수)
- Playwright로 목업 HTML 생성 후 스크린샷
- 날짜·시간·점3개 등 동적 UI 요소 제거 후 촬영
- **디스크립션 항목 수만큼 빨간 원형 번호 배지를 화면 내 해당 UI 요소 위치에 반드시 삽입**
- 배지 번호 ↔ 디스크립션 No 번호 1:1 대응 (파트너/포스앱 모두 동일하게 적용)
- **배지 클리핑 방지 필수**: 카드/컨테이너에 `overflow: hidden` 금지. 대신 `:first-child` / `:last-child`로 개별 border-radius 적용할 것. `overflow: hidden`이 있으면 `top:-11px; left:-11px`으로 튀어나온 배지가 잘림

### 색상 시스템 (Apple 스타일)
| 용도 | 색상 |
|---|---|
| 배경 | `#FFFFFF` (흰색 — 기획서 프레임 및 섹션 배경 모두 흰색으로 통일) |
| 주요 텍스트 | `#1D1D1F` |
| 보조 텍스트 | `#6E6E73` |
| 포인트/배지/버튼 | `#0071E3` |
| 배지 배경 | `#E5EFFF` |

### 텍스트 클리핑 방지 (필수)
- TEXT 노드: `textAutoResize = 'HEIGHT'`
- 텍스트 포함 FRAME: `primaryAxisSizingMode = 'AUTO'`
- 폰트 반드시 `loadFontAsync` 후 수정
