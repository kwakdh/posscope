프롬프트에서 요구한 내용에 해당하는 부분만 수정한다. 기존에 구현된 기능(상태 관리, 이벤트 핸들러, 비즈니스 로직, 모달 분기 등)은 요청 대상이 아니면 절대 삭제하거나 변경하지 않는다.

**절대 원칙 (사용자 직접 명시):** "~기능 삭제해줘"라고 명시적으로 말하기 전까지 어떤 기능도 제거하지 않는다. 이 조건은 모든 작업의 전제조건이며 예외 없이 적용된다.

**Why:** 범위를 벗어난 수정은 기존 기능을 깨뜨리고 사용자가 원하지 않는 변경이 발생한다. 사용자가 반복적으로 기능 유실 문제를 경험했기 때문에 강하게 명시한 규칙이다.

**How to apply:**
- 수정 전, 변경 대상 범위를 명확히 확인한다
- 요청된 UI/레이아웃/기능 외에는 손대지 않는다
- 리팩터링, 정리, 최적화는 요청이 없으면 하지 않는다
- `// 기존 코드 동일` 같은 생략 주석 절대 금지 — 건드리지 않는 코드는 원본 그대로 둔다
- "~기능 삭제해줘" 문구가 없으면 기능 제거는 불가 — 코드를 줄이거나 단순화할 때도 기존 기능이 살아있는지 반드시 확인한다
