POSSCOPE에서 "화면 정책을 딥하게 잡으면서 언제든 동기화되는 편리한 기획 환경"을 만드는 작업의 전체 로드맵.

## 전체 그림
- 이미 배포된 내용: 화면 캡쳐 업로드(+클립보드 붙여넣기) → 정책/디스크립션 작성
- 신규 기능 추가 시:
  - 방식1: 화면 캡쳐 직접 업로드
  - 방식2: 클로드코드에게 "A 메뉴에 기획서 써줘"라고 요청하면 캡쳐본+디스크립션+정책이 자동으로 채워짐
    - 디스크립션/정책은 클로드코드가 Supabase에 직접 write 가능 (실시간 동기화)
    - 와이어프레임 이미지는 클로드코드가 HTML 목업 생성 후 캡쳐 스킬로 스크린샷 → 업로드하는 방식으로 검토
  - 작성된 디스크립션/정책은 POSSCOPE 화면에서 직접 수정 가능, 수정 시 서버에 동기화 저장
  - 신규 기능 추가 시 해당 내용을 NHN 두레이에 일감(이슈)으로 자동 등록

## 진행 상태 (2026-06-15 기준)
- [완료] 캡쳐 붙여넣기 업로드: [feature-detail.tsx](../../../../Desktop/클로드/src/components/feature-detail.tsx)의 와이어프레임 영역에 Ctrl+V 붙여넣기 지원 추가 (`handlePaste`, `uploadFile` 공통화)
- [보류] 두레이 일감 자동 등록: API 토큰/프로젝트 ID 등 접근 정보 미확보. 두레이 연동 작업 시작 전 사용자에게 API 토큰/프로젝트 ID 확인 필요
- [미진행] 클로드코드 "기획서 써줘" 자동화 플로우 (디스크립션/정책 write + 와이어프레임 목업 생성)
