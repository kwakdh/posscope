## 공통 시작점
"결제 요청" → 4개 결제 방식(카드/현금/간편/분할)으로 분기

## 1. 카드 결제
- 단말기(리더기) 연결 유무 (조건)
  - 실패 시: 실패 팝업 "결제기가 실패하였습니다. 카드 단말기에서 연결을 확인해 주세요." → 장비 설정 이동 → 카드 리더기 설정
  - ⚠️ 이슈 메모(빨간 글씨): "단말기에서는 입력 되는데 리더기에서는 입력 요청 확인 불가"
- → 기존 요청 종료 확인 (조건) — 실패 시: 실패 팝업 "중복 거래"
- → 5만원 이하 거래 (조건)
  - → 할부 선택 → 할부 개월 수 입력
  - → 카드 삽입
- → 60초 이내 결제 (조건) — 실패 시: 실패 팝업 "타임아웃"
- → 5만원 이하 거래 (조건) → 서명 패드 출력 → 결제 완료

## 2. 현금 결제
- 받은 금액 입력 (항목: 결제 금액 / 받은 금액 / 거스름 돈)
- → 받은 금액 >= 결제 금액 (조건)
  - Y → 결제 완료 → 현금 영수증 신청 → 현금영수증 정보 입력 → 발행 완료
    - 부가 설명: "핀패드 호출 시: 단말기로 입력 화면 전환 / POS에는 현금 영수증 발급 요청 팝업 출력 / 약 60초 대기", "자진 발급 선택 시: 키패드 비활성화"
  - N → 결제 버튼 비활성화 → 메세지 "결제 금액 보다 받은 금액이 작습니다"
- 옆에 빨간 메모: "간편결제 바로 써보기 → 간편결제 신청 기능 유무 확인" (현금/간편결제 영역 사이에 위치한 별도 메모)

## 3. 간편 결제
- 카메라 스캔 또는 스캐너 사용(카드 단말기 호출) → 리더기 출력 → QR코드 or 바코드 스캔
- → 5만원 이하 거래 (조건) → 할부 선택 → 할부 개월 수 입력 → 서명 패드 출력
- → 가맹점 여부 (조건) — 실패 시: 실패 팝업 "미등록 가맹점"
- → 결제 완료

## 4. 분할 결제
- 분할 결제 금액 입력 → 결제 (카드, 현금, 간편결제, 분할결제 모두 가능)
- → 남은 결제 금액 유무 (조건)
  - N → 결제 완료
  - Y → 결제 내역 보기 → 결제 중단 → 결제 이어하기 (조건)
    - Y → 분할결제 이어하기 → (남은 금액) → 다시 "결제" 단계로 루프
    - N → 결제 완료
  - "남은 금액" 표시는 결제 단계로 직접 루프되기도 함

## Why
실패 팝업/메세지/조건 분기까지 상세히 기획되어 있어, 추후 결제 화면 검토·QA 체크리스트·관련 기획 보강 시 이 구조를 기준으로 삼으면 됨. 특히 "단말기 vs 리더기 입력 확인 불가" 이슈는 미해결 메모로 남아있어 추후 확인 필요.
